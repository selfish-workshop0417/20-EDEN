# taste-archive PRD

## 1. 제품 개요

**제품명:** taste-archive  
**한 줄 설명:** 위시리스트 저장부터 취향 기반 추천, 연예인 착용 아이템 대안 탐색까지 — 나만의 취향 아카이브

### 핵심 가치
- 흩어진 위시리스트를 한 곳에 모아 관리
- 저장된 아이템에서 취향을 학습해 새로운 상품 발견
- 비싸거나 품절된 연예인 착용 아이템의 현실적인 대안 제시

---

## 2. 문제 정의

| 사용자 불편 | 현재 해결 방식 | 한계 |
|---|---|---|
| 위시리스트가 인스타 저장, 네이버 즐겨찾기, 카카오톡 등에 분산 | 수동으로 링크 복사·저장 | 검색 불가, 맥락 없음 |
| 취향에 맞는 상품을 찾는 데 시간 낭비 | 무한 스크롤, 키워드 검색 | 피로도 높음, 발견 낮음 |
| 연예인 착용 아이템이 고가이거나 품절 | 직접 유사 상품 검색 | 시간 소요, 결과 불확실 |

---

## 3. 사용자

### 주 타겟
- **패션 관심 MZ (20–35세):** 연예인·인플루언서 스타일을 참고하고 합리적 대안을 찾는 사용자
- **온라인 쇼핑 헤비유저:** 여러 플랫폼에 걸쳐 위시리스트가 흩어져 있는 사용자

### 사용자 여정
```
발견 → 저장 → 탐색 → 대안 찾기 → 구매
```

---

## 4. 핵심 기능

### 4.1 위시리스트 저장
- URL 붙여넣기 또는 공유 시트로 상품 저장
- 상품명, 가격, 이미지, 브랜드 자동 파싱
- 태그·컬렉션으로 분류
- 가격 변동 알림

### 4.2 취향 기반 상품 추천
- 저장 아이템에서 스타일 벡터 생성 (색상, 카테고리, 브랜드, 가격대)
- 저장·조회·클릭 행동으로 선호 지속 업데이트
- 피드 형태로 개인화 추천 노출
- "이것과 비슷한 상품" 아이템 레벨 추천

### 4.3 연예인 착용 아이템 대안 제시
- 연예인 이름 또는 착용 사진 입력
- 착장 아이템 인식 및 정보 표시 (원본 상품, 가격, 브랜드)
- 유사 스타일 대안 상품 리스트 (가격대별 필터)
- 대안 상품 바로 위시리스트 저장

## 5. 기능 우선순위

| 우선순위 | 기능 | 목표 지표 |
|---|---|---|
| P0 | 위시리스트 저장 (URL 파싱) | 저장 성공률 > 90% |
| P0 | 컬렉션 관리 (생성·수정·삭제) | 리텐션 D7 > 40% |
| P1 | 취향 기반 추천 피드 | 피드 CTR > 15% |
| P1 | 연예인 착장 아이템 검색 | 검색→저장 전환율 > 20% |
| P2 | 가격 변동 알림 | 알림→방문율 > 30% |
| P2 | 소셜 공유 (컬렉션 공유) | 공유 전환율 > 10% |

## 6. 비기능 요구사항

- **URL 파싱 지원 쇼핑몰:** 무신사, 29CM, 쿠팡, 네이버쇼핑, 지그재그, 에이블리 (v1 기준)
- **응답 속도:** URL 파싱 < 3초, 추천 로딩 < 1초
- **데이터 보안:** 위시리스트 기본 비공개, 사용자 공유 선택
- **플랫폼:** iOS + Android (웹 뷰 포함), 웹 대응은 v2
